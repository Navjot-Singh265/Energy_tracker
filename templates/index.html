A* Search — Optimal Appliance Schedule
Problem modelled as State Space Search. State: {appliance → time slot}. Goal: minimise total electricity cost. Heuristic h(n): max possible saving from remaining unscheduled appliances (admissible). A* is guaranteed to find the optimal solution. Constraint Satisfaction handles hard user constraints.
Constraint Satisfaction (CSP)
Hard constraints are enforced via backtracking + forward checking before A* is executed. Soft constraints are handled as weighted cost penalties.
Hard Constraint 1
AC must run 10 AM – 6 PM
Hard Constraint 2
Total load < 5 kW simultaneously
Soft Constraint 1
Washing ≤ once per day
Soft Constraint 2
Water heater before 7 AM only

Responsible AI & Ethics
All LLM interactions follow these responsible AI principles enforced in the Python backend.
Data Privacy: No personal identifiers sent to LLM. Only aggregated, anonymised usage numbers are included in the system prompt.
Transparency: The exact prompt sent to the AI is visible above. You can see every piece of context the model receives.
Bias Mitigation: Responses are grounded in PSPCL Punjab official tariff data to prevent inaccurate regional advice.
User Control: Chat history can be cleared anytime. The LLM is never given previous sessions — each session is fresh.
